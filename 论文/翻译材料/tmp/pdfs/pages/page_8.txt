and context of the incoming question.
During the second stage, Response Similarity Evaluation, the quality 
of the generated response is checked against the real response in the 
selected prompt-response pair. The goal of this verification is to check 
how the generated response matches the expected behavior of the API as 
recorded in the reference dataset. Through the similarity calculation of 
the generated and real response to the prompt-response pair, the system 
ensures that the generated output truly reflects valid API behavior. High 
scores in similarity represent more exact congruence, implying that the 
decoy API is indeed mimicking the real responses. The process grows 
credibility for the honeypot, which is effective in logging realistic in -
teractions with attackers and deriving valuable information on potential 
threats.
6.2. Results
We evaluate the performance of the decoy web API honeypot in 
generating realistic and contextually accurate responses. The evaluation 
is based on a two-step similarity analysis, involving (1) the identification 
of the most similar prompt-response pair for each incoming request and 
(2) the comparison between the generated response and the actual 
response associated with the matched prompt-response pair. Table 1
provides a summary of sample requests, detailing the request similarity 
scores, reference responses, generated responses, and response similar -
ity scores.
For every incoming request, the system calculates a Request Simi -
larity score, matching it against previously stored ones in the dataset, to 
select the prompt-response pair having the highest degree of match. This 
selection shows the systems ’ ability to correctly single out contextually 
relevant prompts. As a case in point, the GET request with a similarity 
score of 0.9887 was matched with a saved request that fetches trans -
action details, demonstrating that the system was able to recognize and 
align the context of the request in the dataset accurately. The Response 
Similarity score quantifies how much the generated response corre -
sponds to the reference response from the relevant prompt-response 
pair, ensuring that the simulated API functionality in the context of 
IoT home automation is accurate. The system correctly emulates ex -
pected API outputs, as evidenced by the similarity scores of 1.0000 for 
POST adding a new smart device and 0.9992 for DELETE removing a 
user ’ s profile. These results confirm this system in its capability of 
creating realistic yet contextually exact decoy responses to impersonate 
real API interactions aimed at deceiving attackers and gathering 
meaningful intelligence.
An even better illustration of the honeypot ’ s efficacy in response 
generation is demonstrated by the average response similarity score of 
0.9780. Such a high average score indicates a system with a reliable 
ability to generate responses that closely match the expected results, 
which would help it serve more effectively as a powerful deception 
mechanism in cybersecurity. By generating replies that are plausible and 
contextually relevant, the decoy web API can go much deeper in its 
ability to interact with potential attackers, thus revealing a great deal 
about their tactics and motivations.
7. Conclusion
This work describes the design and evaluation of a decoy Web API 
honeypot based on state-of-the-art language models with Retrieval- 
Augmented Generation, for Deriving believable answers and enlarging 
its potential of being an effective decoy in the context of cybersecurity. 
The architecture is able to obtain contextually appropriate and plausible 
responses by adaptively matching incoming requests to the established 
prompt-response pairs and evaluating response accuracy by applying 
similarity metrics.
These results show that the honeypot is capable of actively inter -
acting with potential attackers, as indicated by the high request and 
response similarity scores. With a mean response similarity value of 
0.9780, the system proves capable of closely mimicking expected API 
behaviors, thereby rendering an authentic user experience that en -
courages a longer stay by users. It has a high level of realism, which not 
only deceives the adversaries but also allows gathering important in -
sights about their strategies and methodologies for improving defensive 
capabilities. This research delineates the potential benefits of using LLM- 
based honeypots in the cybersecurity domain to protect critical systems 
by redirecting aggressors away from actual resources and monitoring 
their actions in a controlled environment. Future work will continue to 
diversify the dataset of prompt-response pairs, allowing for a larger 
range of API interactions. Another promising avenue is increasing the 
model ’ s real-time adaptability in order to detect even more complex 
forms of malicious behavior. Therefore, integrating state-of-the-art 
artificial intelligence into honeypot systems is considered a very prom -
ising direction forward, shaping the future of advanced and proactive 
cyber defense solutions.

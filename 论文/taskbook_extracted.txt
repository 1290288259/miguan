一、毕业设计（论文）的内容
在数字化时代，网络攻击手段日趋隐蔽和多样化，传统被动防御方式难以有效应对未知威胁。蜜罐技术作为一种主动防御手段，通过模拟脆弱的网络服务吸引攻击者，可实现对恶意流量的主动捕获、分析与防御，为网络安全防护提供关键威胁情报。基于此，本设计旨在构建一套轻量化的小型蜜罐恶意流量捕获和防御系统，集蜜罐部署、流量监控、威胁响应与数据可视化于一体，帮助用户直观了解网络攻击态势并实施有效防御。具体设计内容如下：
（1）登录模块：实现用户身份认证功能，支持管理员账户的注册、登录及权限管理。通过密码加密存储（采用 SHA-256 算法）和登录失败次数限制（如 5 次失败后临时锁定），确保系统访问的安全性，防止未授权用户篡改蜜罐配置或查看敏感日志。
（2）流量日志模块：实时记录所有访问蜜罐的恶意流量信息，构建结构化日志体系。日志字段包括攻击源 IP、攻击目标服务、攻击行为特征（如SQL 注入语句）、攻击时间戳、连接状态等，并存储于MySQL数据库中，支持日志的查询、筛选与导出，为攻击分析提供原始数据。
（3）IP 封禁模块：基于流量日志分析结果执行自动或手动防御动作。支持自定义封禁规则（如单 IP 单日攻击次数超过阈值自动封禁），通过调用系统底层命令阻断恶意 IP 的网络连接，并维护动态黑名单，允许管理员手动添加 / 移除封禁 IP，实现对高频攻击源的精准拦截。
（4）蜜罐部署模块：提供低交互式蜜罐服务的可视化配置与管理功能。支持部署多种模拟服务，用户可通过界面设置服务端口、响应信息等参数，控制蜜罐的启动/停止状态，并实时监控各服务的运行状态。
（5）数据可视化模块：将捕获的恶意流量数据转化为直观的图表展示。通过攻击趋势折线图（展示每日攻击次数变化）、攻击源地理分布图（基于 IP 属地解析）、服务攻击热力图（统计各蜜罐服务被攻击的频率）、TOP10 恶意 IP 排行榜等，帮助用户快速掌握网络攻击态势，为防御策略调整提供决策支持。
二、毕业设计（论文）的要求与数据 
1. 要求:
（1）用户管理：实现用户注册、登录、信息管理支持用户个人信息的编辑与查看，基于 JWT（JSON Web Token）实现身份认证，确保会话安全。
（2）蜜罐管理：构建可配置的蜜罐服务体系，实现蜜罐的启动 / 停止控制及状态监控（在线 / 离线），满足多样化恶意流量捕获需求。
（3）流量处理与防御：实时捕获并解析蜜罐流量，提取攻击源 IP、行为特征等关键信息；支持手动添加 / 移除黑名单，通过调用系统命令实现网络阻断，确保防御响应的及时性。
（4）架构设计：采用前后端分离架构，前端基于 Vue 框架构建交互式界面，后端使用 Python的Flask框架提供API服务，通过RESTful接口实现数据交互，确保系统模块解耦、扩展性强。
（5）轻量化部署：无需复杂容器化，基于Python虚拟环境和Vue打包部署，小型服务器上运行，简化环境配置，降低部署门槛，适合小型网络场景使用。
（6）性能测试：模拟多源IP并发攻击场景，测试系统在高并发下的流量捕获效率、日志存储速度及防御响应延迟，确保核心功能（如攻击识别、IP 封禁）在流量峰值时仍能稳定运行。
（7）安全性：对敏感数据（如用户密码）采用 SHA-256 加密存储，实现登录频率限制、异常IP访问告警，防止系统自身被恶意攻击，保障平台运行安全。
2. 所需数据:
（1）用户数据：用户名、加密后的密码、角色类型、注册时间、最近登录时间、操作日志（如部署蜜罐、封禁 IP 的记录）。
（2）蜜罐配置数据：蜜罐 ID、服务类型、监听端口、启动状态、创建时间、响应信息模板。
（3）流量日志数据：攻击事件ID、源IP地址、攻击目标蜜罐、攻击行为描述（如注入语句）、攻击时间戳、连接持续时长、是否触发封禁。
（4）封禁数据：黑名单IP、封禁时间、累计封禁次数、触发原因（如 “1 小时内攻击 10 次”）、解封状态（手动）、操作人（系统自动/管理员）。
3. 所需条件:
（1）熟悉蜜罐技术原理及常见网络攻击类型，了解 TCP/IP协议及Socket网络编程。
（2）具备数据库设计基础，能够使用MySQL管理用户数据、日志数据及配置信息，掌握数据查询与优化技巧。
（3）精通 Python（Flask框架）进行后端 API开发，熟悉Vue.js框架实现前端界面，能够独立完成前后端数据交互与状态管理。
（4）了解操作系统命令，具备基本的网络配置与安全防护知识，能够测试并验证 IP 封禁功能的有效性。
